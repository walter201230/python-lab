# 三、主模块和非主模块

## 1、定义

- **主模块**：被 **直接运行** 的那个 `.py` 文件
- **非主模块**：被别的模块 `import` 进来的那个

类比：你 `python a.py`，`a` 就是主模块；`a.py` 里如果 `import b`，`b` 就是非主模块。

## 2、`__name__` 属性

怎么在代码里判断 **当前模块到底是被直接运行还是被别人 import 的**？

Python 给每个模块都设置了一个特殊变量 `__name__`：

- 模块被 **直接运行** 时，`__name__` 的值就是字符串 `'__main__'`
- 模块被 **import** 时，`__name__` 的值就是模块名本身（比如 `'mymod'`）

所以经典写法是：

```python
# mymod.py

def main():
    print('正在运行 main 逻辑')

if __name__ == '__main__':
    main()
```

效果：

- 直接 `python mymod.py` → 打印 `正在运行 main 逻辑`
- 别的文件 `import mymod` → 不会自动跑 `main()`，函数依然可以被调用

> 这是 Python 中最常见的一段「魔法咒语」。它的意义是：**写一个模块时，既能让它当库被别人 import，又能让它能直接被运行做测试或当脚本用**。

## 3、为什么要这么写？

不写 `if __name__ == '__main__':` 守卫的话，模块里所有顶层代码 **都会在被 import 时执行**——往往不是你想要的。

```python
# bad_mymod.py
def add(a, b):
    return a + b

print(add(1, 2))   # 这一行会在被 import 时立刻执行
```

```python
# 别的文件
import bad_mymod   # 副作用：屏幕上立刻打印了 3
```

加上 `if __name__ == '__main__':` 包裹后，这个 `print(add(1, 2))` 就只在直接运行时执行，被 import 时不会触发。
