# 三、`__eq__`：自定义「相等」的含义

默认情况下，`==` 比较两个对象是不是**同一个对象**（同一块内存），等价于 `is`：

```python
class Money:
    def __init__(self, amount):
        self.amount = amount

a = Money(100)
b = Money(100)
print(a == b)  # False  ← 两个不同对象，默认就是不相等
```

但从「业务」角度看，100 块钱就是 100 块钱——只要金额相同，就该相等。这时候定义 `__eq__`：

```python
class Money:
    def __init__(self, amount):
        self.amount = amount

    def __eq__(self, other):
        return self.amount == other.amount

a = Money(100)
b = Money(100)
c = Money(50)

print(a == b)  # True
print(a == c)  # False
print(a != c)  # True   ← != 默认是 __eq__ 的反
```

`__eq__(self, other)` 接收两个参数，返回 `True` / `False`。Python 会把 `a == b` 翻译成 `a.__eq__(b)`。

## 完整的比较运算符家族

| 写法 | 魔法方法 |
|---|---|
| `a == b` | `__eq__` |
| `a != b` | `__ne__`（不写也会自动用 `__eq__` 取反）|
| `a < b` | `__lt__` |
| `a > b` | `__gt__` |
| `a <= b` | `__le__` |
| `a >= b` | `__ge__` |

举个完整例子：

```python
class Number:
    def __init__(self, value):
        self.value = value

    def __eq__(self, other):
        return self.value == other.value

    def __lt__(self, other):
        return self.value < other.value

n1 = Number(2)
n2 = Number(3)
print(n1 == n2)  # False
print(n1 < n2)   # True
```

## `__hash__` 的小坑

只要你重写了 `__eq__`，Python 会**自动把 `__hash__` 设为 None**——也就是说，这个对象不能再放进 `set` / `dict` 的 key。

如果想让对象既可比较又可哈希，要手动加上 `__hash__`：

```python
class Money:
    def __init__(self, amount):
        self.amount = amount

    def __eq__(self, other):
        return self.amount == other.amount

    def __hash__(self):
        return hash(self.amount)
```

> 规则：相等的对象，hash 必须相等。所以 `__hash__` 通常基于 `__eq__` 用到的同一组属性。
